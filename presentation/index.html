The Problem

Security teams are drowning in alerts

  • Mean time to patch: weeks
  • Air-gapped systems require on-premise tooling
  • Existing SOC tools detect → they don't fix
Demo voiceover: "Security teams face thousands of alerts daily. They can detect threats, but patching the underlying vulnerability still requires a human to trace code, write a fix, test, and deploy."

The Solution

┌─────────────┐ │ Network │ │ Traffic │ └──────┬──────┘ │ ┌──────▼──────┐ │ LogMonitor │ Agent 1: Detect │ (alert) │ └──────┬──────┘ │ AlertEvent ┌──────▼──────────┐ │ ForensicInv. │ Agent 2: Investigate │ (LLM + source) │ ↓ LLM or heuristic └──────┬──────────┘ │ ForensicReport ┌──────▼──────────┐ │ PatchEngineer │ Agent 3: Patch │ (LLM + ast) │ ↓ LLM or lookup └──────┬──────────┘ │ PatchProposal ┌──────▼──────────┐ │ Sandbox Test │ Validate │ (harness) │ ↓ pass/fail └─────────────────┘

Agent 1: Log Monitor

Tails traffic.log in real time

  • Detects 15+ suspicious patterns (echo, whoami, &&, etc.)
  • Emits structured AlertEvent
  • Severity classification (info → critical)

{
  "alert_id": "a1b2c3d4e5f6",
  "endpoint": "/execute",
  "payload": "dir src && whoami",
  "severity": "high",
  "suspicious_indicators": ["dir", "whoami", "&&"]
}

Agent 2: Forensic Investigator

LLM primary → heuristic fallback

  • Sends full source file + raw logs to LLM
  • LLM pinpoints exact vulnerable file & line
  • Post-hoc source-line validation catches wrong answers
  • Falls back to keyword heuristic if confidence < 0.7

{
  "file": "src/sandbox_target/app.py",
  "line": 34,
  "vuln_type": "command_injection",
  "confidence": 0.95
}

Agent 3: Patch Engineer

LLM primary → lookup fallback

  • LLM generates secure code replacement
  • ast.parse() validates syntax (with dedent)
  • Up to 3 retries with syntax error feedback
  • Hardcoded shlex.split() lookup as safety net
  • Applies patch + triggers hot-reload

# BEFORE                         # AFTER
subprocess.run(                  subprocess.run(
    cmd,                             shlex.split(cmd),
    shell=True,                      capture_output=True,
    capture_output=True,             text=True,
    text=True,                       timeout=5,
    timeout=5,                   )
)

Key Architecture Decisions

  • Hardware-agnostic core — ROCm is a drop-in endpoint swap
  • LLM-first, not AI-only — 3 safety nets (confidence, source validation, heuristic)
  • Strict I/O contracts — Pydantic schemas enforce structure between agents
  • Self-correction — syntax errors fed back into LLM for regeneration
  • Isolated testing — sandbox harness validates every patch before deploy
